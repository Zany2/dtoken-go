# API 稳定性说明

这个项目是模块化设计，但公开 API 会尽量在补丁版和小版本中保持稳定。

## 稳定的公开区域

- `core`：核心认证与管理逻辑。
- `dtoken`：主全局门面。
- `defaults`：默认构建器与内置组件装配。
- `com/*`：可复用的组件实现。
- `integrations/*`：各框架适配层与中间件。
- `sso/*`：可选的 SSO 相关模块。

## 需要保持兼容的内容

- 公开函数名称和参数签名。
- 框架适配器和示例会直接使用的公开类型。
- 公开 API 返回的错误值。
- 已发布 tag 对应的模块路径和包结构。

## 可以调整的内容

- `internal` 路径下的内部辅助代码。
- 未导出的字段和函数。
- 不属于 API 合同的示例代码与文档。

## 版本规则

- 补丁版本不应破坏现有调用方。
- 小版本可以增加新 API，但应尽量不破坏旧 API。
- 破坏性变更只应在明确升级版本时引入。

包级使用细节可参考 [Core API 速查](core-api-cheatsheet_zh.md)。
